# Ciclo de vida 
## Planeación
- Se asignan los roles de Scrum 
- Establecemos la duracion de cada entregable o sprint segun requiera 

## Analisis
¿Que debe hacer exactamente el software?
- Determinamos los requisitos funcionales creando las  *Historias de Usuario* en base a los perfiles del Junior y el Senior 
-  Definimos requisitos tecnicos, es decir, que tecnologias web usaremos (HTML5, CSS3, JavaScript vanilla o frameworks)
## Diseño 
Implementaremos la estructura del sitio web 
- Definimos la estructura del Backlog en Trello 
- Diseñamos la interfaz segun la distrubucion visual que se defina 
- Implementamos la representacion de los videos en el codigo
## Implementacion
Escribimos el código fuente del software y coordinamos como lo trabajaremos en equipo

- El *Scrum Master* genera el repositorio en GitHub y añade a los integrantes al entorno de desarrollo
- Cada integrante clona el repositorio localmente, abre su entorno y ejecuta git checkout -b rama-su-nombre para aislar su espacio de trabajo y evitar pisar el código de sus compañeros.
- Cada programador implementa su archivo específico desde su rama para luego subirlo a la nube mediante git push origin rama-su-nombre.
## Testing o Pruebas 
- Antes de aprobar el merge, el equipo valida que los archivos individuales creados por cada integrante convivan correctamente en la rama principal sin generar conflictos de Git.
- Se comprueba de manera funcional que los filtros respondan correctamente al hacer clic y que los enlaces de los videos no estén caídos.

## Mantenimiento
Se pone el software a dispocision de los usuarios 
- Para la entrega formal del trabajo, se consolida la rama main actualizada con todos los archivos de código integrados
